\section*{Part 1: Summary of Today's Work (PySR Integration \& Experiments)}

Today's focus was implementing and verifying the symbolic regression experimental benchmarking module in \texttt{src/experiments.py} and its unit tests in \texttt{tests/test\_experiments.py}. This establishes the $S(i,j)$ data generation layer.

\begin{itemize}
    \item \textbf{Layer 1 -- Dataset Loaders:} Implemented deterministic loaders (\texttt{load\_feynman\_ke}, \texttt{load\_feynman\_coulomb}, \texttt{load\_polynomial}, \texttt{load\_srsd\_dummy}) with an 80/20 train/test split. Seed-locked dataset splits isolate search randomness from partition variance.
    \item \textbf{Layer 2 -- PySR Kwargs Mapper \& Composer:} Designed and implemented \texttt{build\_pysr\_kwargs} to map active constraints (\texttt{C1}--\texttt{C4}) dynamically to PySR 1.5.10 regressor parameters. Composed \texttt{nested\_constraints} (trig nesting depth), \texttt{maxdepth}, whitelisted operators (by set intersection), and \texttt{elementwise\_loss} custom Julia-based penalties additively.
    \item \textbf{Layer 3 -- Scenario Runner:} Implemented \texttt{run\_scenario} to run serial deterministic PySR fits, record search costs (via wall-clock timing as the v1.5.10 proxy), calculate recovery metrics (MSE, relative MSE, NED), and run post-hoc constraint checks on the best expression.
    \item \textbf{Layer 4 -- Matrix Orchestrator:} Implemented \texttt{run\_experiment\_matrix} to execute the $4 \times 12 \times 5 = 240$ matrix, save run environment hashes/git metadata in \texttt{metadata.json}, and log results incrementally to a run-specific CSV file.
    \item \textbf{Testing:} Added 23 new tests in \texttt{tests/test\_experiments.py} covering loader shapes/determinism, ground-truth recovery precision, kwargs mapping/merging, and Levenshtein fallback. The entire test suite has \textbf{83 passing tests}.
    \item \textbf{Orchestrator Validation:} Successfully ran a mini-matrix integration test, verifying that \texttt{metadata.json} captures the environment correctly and the results CSV writes and flushes row-by-row.
\end{itemize}

\section*{Part 2: Strict Plan for Tomorrow (Analysis \& Validation)}

With raw $M(i,j)$ (density estimates) and $S(i,j)$ (search costs) data generators fully complete, tomorrow's plan is to tie the components together to validate the project's primary acceptance criterion.

\begin{itemize}
    \item \textbf{Phase 1 -- Implement \texttt{src/analysis.py}:} Build the analysis module to:
    \begin{itemize}
        \item Load the raw \texttt{pysr\_results.csv} files.
        \item Compute $S(i,j) = \min(S_i,\, S_j) / S_{ij}$ from observed wall-clock times.
        \item Exclude timeout-censored runs from $S(i,j)$ calculations using the \texttt{timeout\_hit} column.
        \item Read $M(i,j)$ matrices from the Monte Carlo runs.
        \item Calculate Pearson $r$ and $p$-values between $M(i,j)$ and $S(i,j)$.
    \end{itemize}
    \item \textbf{Phase 2 -- Acceptance Verification:} Run the full analysis matrix and check if Pearson $r > 0.6$ and $p < 0.05$ holds across the benchmarks, completing the core acceptance criterion.
    \item \textbf{Phase 3 -- Visualisation:} Generate scatter plots of $M(i,j)$ vs. $S(i,j)$ and heatmap visualizations of the interaction matrix.
\end{itemize}
