## 1. Executive Summary

Symbolic Regression (SR) is NP-hard due to its combinatorial search space. Practitioners constrain the space using grammar rules, operator whitelists, depth limits, and dimensional constraints. However, **how pairs of constraints interact — whether synergistically, redundantly, or independently — is not quantified in any existing work.**

This project defines, implements, and empirically validates a **constraint-interaction metric** M(i,j) for SR. The metric predicts whether combining two constraints will yield super-additive search performance gains. The primary success criterion (from Research Plan §1) is:

> **Pearson r > 0.6, p < 0.05** between M(i,j) and the empirical speedup S(i,j), after Benjamini–Hochberg FDR correction.

Deliverables: open-source code repository, a 55–65 page implementation handbook, and results suitable for a workshop or short conference submission.

## 2. Problem Statement

### 2.1 Formal Definition

Let E be the space of expressions generated by grammar G. For a constraint C: E → {0,1}:

```
ρ(C) = Pr_{E~G}[C(E) = 1]
```

The **interaction coefficient** (Research Plan Eq. 1):

```
M(i,j) = ρ(C_i ∧ C_j) / (ρ(C_i) · ρ(C_j))
```

| M(i,j) | Interpretation |
|---|---|
| > 1 | **Synergy** — co-occur more than chance; combining may yield super-additive gains |
| < 1 | **Redundancy** — one subsumes the other; adding the second has diminishing returns |
| ≈ 1 | **Independence** — constraints act on orthogonal dimensions |

### 2.2 Speedup Metric S(i,j)

The empirical validation target (Research Plan §8):

```
S(i,j) = min(evals_{C_i}, evals_{C_j}) / evals_{C_i ∧ C_j}
```

S > 1 means the joint constraint outperforms either constraint alone. M(i,j) must predict S(i,j) with Pearson r > 0.6.

### 2.3 Research Gap

All prior work (Brence et al. 2023, Reissmann et al. 2025, Xiang et al. 2025) studies individual constraint effectiveness. **No existing work models pairwise statistical interaction between constraints as a predictive signal for search performance.** M(i,j) is analogous to "lift" in association rule mining, but applied to a generative grammar distribution — novel in the SR literature.

## 6. System Architecture

Six modules arranged in a pipeline:

```
config.yaml  (single source of truth)
     │
     ▼
[1] expr_generator.py   ← Stochastic grammar-based AST builder
     │  valid sympy.Expr
     ▼
[2] constraints.py      ← Pure Expr → bool constraint factories (C1–C5)
     │  boolean masks
     ▼
[3] metric.py           ← Monte Carlo ρ estimator, M matrix, bootstrap CI
     │  M(i,j) + CIs
     ▼
[4] experiments.py      ← PySR integration, scenario orchestration, S(i,j)
     │  runtime, evals, recovery
     ▼
[5] analysis.py         ← Correlation, regression, FDR correction, plots
     │  figures, tables
     ▼
[6] main.py             ← CLI: --phase={metric,experiment,analysis}
          ↓
      results/<run_id>/
          metadata.json
          interaction_matrix.npy
          pysr_results.csv
```

## 7. Functional Requirements

### 7.1 Expression Generator (`expr_generator.py`) — ✅ IMPLEMENTED

**FR-GEN-01 Grammar Fidelity.** Produce expressions strictly conforming to `config.yaml`:
- BNF: `<expr> ::= <expr> <binop> <expr> | <unop>(<expr>) | <variable> | <constant>`
- `binop ∈ {+, -, *}` (no `/` by default — avoids domain errors; configurable extension)
- `unop ∈ {sin, cos}`, `variable ∈ {x, y}`, `constant ∈ ±{1…9}`

**FR-GEN-02 Two-Pass Validity.** Every returned expression must:
1. Parse via `sympify(evaluate=False)` (preserves AST structure for constraint checking).
2. Return finite real values at `domain_validation.n_test_points` probe points in `test_range`. Catches `zoo`, NaN, ∞, complex results that `evaluate=False` misses.

**FR-GEN-03 Depth Control.**
- `generate_sympy(max_depth=d)` → depth ≤ d (stochastic, early-leaf returns).
- `generate_sympy_at_depth(d)` → depth == d (forced path to depth 0).
- `generate_stratified(N, depths)` → equal quotas per depth bucket {2, 4, 6, 8} to prevent depth bias in density estimates.

**FR-GEN-04 Isolated RNG.** Isolated `random.Random` seeded from `config.yaml → monte_carlo.random_seed`. Must never touch global Python `random` state (breaks reproducibility with PySR/SciPy).

**FR-GEN-05 Stats Contract.** Keys: `generated`, `sympify_failed`, `domain_rejected`, `returned_none`. Callers must `reset_stats()` before each run and log stats after.

---

### 7.2 Constraints (`constraints.py`) — ✅ IMPLEMENTED

**FR-CON-01 Interface Contract.** Each constraint is a pure function `C: sympy.Expr → bool`:
- No mutation of input; deterministic; no shared mutable state.
- Returns `False` (never raises) on edge cases: constants, single-variable, deeply nested.

**FR-CON-02 C1 — Structural.** `make_c1_structural(config)`:
- C1a: max nested trig depth ≤ `structural.max_nested_trig` (default: 1).
- C1b: no subtree with > `structural.max_consecutive_binary` consecutive binary ops (uses `len(args)` due to SymPy flattening: `(a+b)+c → Add(a,b,c)`).
- *Note from Research Plan §5:* C1 constrains **tree shape**; C3 constrains **node labels** — intentionally distinct so M(C1, C3) is non-trivial.

**FR-CON-03 C2 — Depth.** `make_c2_depth(config)`: SymPy `.args`-walk depth ≤ `depth.limit` (default: 6). Bounded divergence ≤+2 vs generator depth (negative constants, subtraction). Measured by `test_depth_divergence_measured`.

**FR-CON-04 C3 — Operator Whitelist.** `make_c3_operator(config)`: all AST nodes in `operator_whitelist.allowed`. Pure node-type inspection; safe against `x**2` vs `x*x` because generator uses `evaluate=False`.

**FR-CON-05 C4 — Positivity.** `make_c4_positivity(config)`: f(x,y) ≥ 0 at **200 random points** in `[-10,10]²`. From Research Plan §5: `P(pass | violates) ≤ (0.95)^200 < 10⁻⁴` if 5% of domain violates. Fast `lambdify` path with `.subs()` fallback. RNG from `positivity.rng_seed`.

**FR-CON-06 C5 — Dimensional (Optional).** Variables carry physical dimensions (x=length, y=time). Propagate units through operations; reject inconsistencies (e.g., length + time). Enabled via `constraints.dimensional.enabled: true`. **Cut first if timeline is short.**

**FR-CON-07 Aggregator.** `build_constraints(config, include_c5=False)` → ordered list `[C1, C2, C3, C4]`. This is the canonical list passed to `DensityEstimator`.

---

### 7.3 Metric (`metric.py`) — ✅ IMPLEMENTED

**FR-MET-01 Density Estimation.** `DensityEstimator.estimate(constraints, N) → RhoResult`:
- `reset_stats()` before drawing; draw exactly N valid expressions via while-loop retry.
- Single pass over N expressions: O(N·k²) total comparisons, not k separate passes.
- Returns `RhoResult`: `rho_i` (k,), `rho_ij` (k,k), `N_actual` (== N always), `stats`.

**FR-MET-02 Interaction Matrix.** `InteractionMatrix.compute(rho_result) → ndarray`:
- `M[i][j] = rho_ij[i][j] / (rho_i[i] * rho_i[j])` for upper triangle.
- **Explicitly mirror** to lower triangle — guarantees M[i][j] == M[j][i].
- M[i][j] = NaN (not raise, not zero) when either ρ = 0.

**FR-MET-03 Bootstrap CI.** `BootstrapCI.compute(expressions, constraints, B=1000)`:
- B=1000 resamples with replacement; 95% CI via percentile method (2.5th, 97.5th).
- Under H₀ of independence: M=1. If 1 ∉ CI, reject independence.
- Accepts `grammar_config` dict for §9 reproducibility snapshotting.

**FR-MET-04 NaN Safety.** All downstream code propagates NaN without crashing.

**FR-MET-05 Calibration Benchmark.** `run_benchmark(generator, N, ...)`: logs stats to `results/benchmark_<N>.json` at N=1k, 10k, 100k. Must **not** assert hardcoded thresholds — calibrate from real output per Research Plan §8.3.

**FR-MET-06 Convergence Check.** Verify `|ρ̂_100k - ρ̂_10k| < 0.01` for each constraint. This is a required validation step, not optional.

### 7.4 Experiments (`experiments.py`) — 🔲 TODO

**FR-EXP-01 PySR Integration.** Run PySR controlled by:
- `pysr.population_size=40`, `niterations=100`, `timeout_in_seconds=300`, `maxsize=25`
- **Determinism**: `deterministic=True`, `parallelism="serial"`, and `random_state=seed` to ensure bit-identical reproduction (NR-REP-01).
- `experiments.repeats=5`, `experiments.seeds=[42,123,456,789,1024]`
- Fallback to `gplearn` if PySR/Julia installation fails.

**FR-EXP-02 Constraint Enforcement in PySR.**
- Structural/operator constraints → PySR `constraints` and `nested_constraints` parameters (hard enforcement during search).
- Positivity → custom `elementwise_loss` penalty (soft enforcement).
- **Composition**: Joint scenarios deeply merge constraint dictionaries and intersect operator whitelists.
- **Post-hoc Compliance**: Best expression is passed through `constraints.py` functions to record true compliance.

**FR-EXP-03 Datasets.**
- **D1 Feynman-1**: Kinetic energy `KE = 0.5·m·v²`
- **D2 Feynman-2**: Coulomb `q1·q2/(4πε·r²)`
- **D3 Polynomial**: Synthetic 3-term polynomial
- **D4 SRSD**: Dataset with dummy variable (Matsubara et al., 2024) — tests whether constraints correctly prune irrelevant variables.

**FR-EXP-04 Scenario Matrix.** For each dataset:
- **Baseline**: No constraints.
- **Singles**: Each of C1–C4 individually (4 runs).
- **Key Pairs**: Pairs with extreme predicted M — highest synergy and highest redundancy (focus resource here).
- **All**: C1–C4 combined.
- 5 repeats per scenario; fixed seeds from config.

**FR-EXP-05 Metrics Recorded Per Run.**
- Wall-clock runtime (seconds). **Note: PySR v1.5.10 does not expose candidate evaluation counts. Wall-clock time is the sole search-cost proxy for S(i,j).**
- Recovery rate (binary): NED < 0.1 **or** relative MSE < 10⁻⁶ on 20% held-out split. **This is the primary recovery criterion.** Symbolic equivalence via `check_semantic_exact()` (SymPy simplification) is available in `analysis.py` as a **secondary, exploratory** enrichment column (`semantically_exact: True/False/None`) but is NOT used to determine `recovered`. Rationale: SymPy's `simplify()` is unreliable for complex expressions (hangs, false negatives on equivalent-but-unsimplifiable forms). Numerical MSE on held-out data is the operationally reproducible criterion.
- Solution complexity (AST node count of best expression).
- Timeout censorship: If the timeout is hit, record `recovery=False` and `wall_clock_s = timeout`. These must be filtered from S(i,j) regression.
- Best expression string.
- **HoF compliance audit**: `hof_c1_violation_rate`, `hof_c2_violation_rate` — post-hoc constraint violation rates across the full Pareto front, quantifying the M(i,j) vs S(i,j) distribution gap.

**FR-EXP-06 Reproducibility Logging.** Write `results/<run_id>/metadata.json` **before** run starts:

| Field | Source |
|---|---|
| Git commit hash | `git rev-parse HEAD` |
| Config SHA-256 | `hashlib.sha256(config_bytes)` |
| Random seed | `config.yaml → experiments.seeds` |
| Sample size N | `config.yaml → monte_carlo.N` |
| Grammar version | full `config.yaml → grammar` dict |
| Timestamp UTC | `datetime.utcnow().isoformat()` |
| Python version | `sys.version` |
| Package versions | sympy, numpy, pysr pinned |

**FR-EXP-07 Append-Only.** Never overwrite previous runs. Each run → unique `run_id` (timestamp + seed).

### 7.5 Analysis (`analysis.py`) — 🔲 TODO

**FR-ANA-01 Speedup Computation.** For each (i,j) pair and dataset:
```
S(i,j) = min(evals_{C_i}, evals_{C_j}) / evals_{C_i ∧ C_j}
```

**FR-ANA-02 Correlation Analysis.**
- Pearson and Spearman correlation between M(i,j) and S(i,j).
- Linear regression: `S(i,j) = a + b·M(i,j) + ε`. Significant positive b (p < 0.05) confirms predictive value.

**FR-ANA-03 Multiple Comparisons.** Apply Benjamini–Hochberg FDR correction at α=0.05 across all pairwise tests.

**FR-ANA-04 Power Analysis.** With C(4,2)×4 = 24 (pair, dataset) data points: power analysis confirms detectability of r ≥ 0.6 at α=0.05 with power > 0.80 (`statsmodels.stats.power`).

**FR-ANA-04a Power Under Attrition.** `load_and_preprocess` censors non-converged runs (timeout_hit & ~recovered → NaN). Power under realistic attrition:

| Attrition | N_eff | Power (r=0.6) | Adequate? |
|---|---|---|---|
| 0% | 24 | 0.940 | ✅ |
| 10% | 21 | 0.905 | ✅ |
| 20% | 19 | 0.871 | ✅ |
| 30% | 16 | 0.801 | ✅ |
| 40% | 14 | 0.737 | ❌ |

**Commitment:** If attrition exceeds 30% (N_eff < 16), the study cannot achieve 0.80 power at r=0.6. In this case, we will report the observed r and its bootstrap CI without making a binary significance claim.

## 12. Known Gaps & Open Questions

**Known Gaps (M ≠ S Divergences to document in Ch. 13 and Ch. 9):**
1. **C1b not enforced in PySR**: PySR has no parameter for sibling count limits in flattened nodes. PySR search only restricts C1a (nested trig).
2. **C2 depth conventions**: PySR's `maxdepth` acts on internal binary trees, which differs from SymPy `.args` depth used for the density estimate.

**Open Questions:**
1. **C5 Dimensional**: `constraints.dimensional.enabled: false`. Include in interaction matrix or skip entirely? Adds O(k) complexity to experiments.
2. **Grammar sweep**: Should the manuscript quantify how M changes when grammar changes (add `/`, `exp`)? Requires multiple grammar runs — likely out of scope for this timeline.
3. **C4 Loss Penalty**: What is the optimal multiplier for the `elementwise_loss` soft penalty to balance constraint satisfaction without stifling search?
4. **Bootstrap p-value for M ≠ 1**: Report formal H₀: M=1 test alongside CIs, or CIs only? Adds clarity but increases reporting burden.
5. **Figures in git**: Commit `figures/` as binary artifacts, or re-generate on demand from `results/`? Latter is cleaner but requires stable plotting script.
